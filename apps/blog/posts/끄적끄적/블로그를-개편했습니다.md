### 글은 한번만 쓸래

막상 포폴을 완성하니 엄청 뿌듯했다.
나는 당시에  [Velog](https://velog.io/@oilater/posts)에 공부한 내용들(TIL)을 꾸준히 쓰고 있었는데, 포폴에서도 게시물을 보여주고 싶었다. 
그런데 포스트를 양쪽에서 함께 관리하는 건 딱봐도 귀찮아보였다.

마침 FE 지인이 NotionPresso라는 라이브러리를 만들고 있길래
'그럼 나는 Velog를 띄워야지' 생각이 들어서 스타일링했다.
모든 구현을 마치고나서야 MDX라는 것을 알게 되었다,, ㅠㅠ

### 불편했던 점들

1. 내 글 가져오려고 GraphQL, Tanstack을 쓰는 것부터 좀 짜쳤다. 포스트들을 빠르게 보여주기 위해 마크다운 파싱 로직을 서버로 분리했다. 하지만 무한 스크롤로 포스트를 가져오려면 클라이언트에서 외부 API 서버로 요청을 보내야하는데, 이때 CORS 오류를 만나게 된다. API Route를 거치면 속도가 체감될 정도로 느려져 rewrites 프록시를 적용하는 등 신경 쓸 부분이 많았다. 그래도 배우는 건 많았다.

2. 포스트를 카테고리별로 분류하기가 쉽지 않다. 코드가 복잡해지는 게 싫어서 결국 그냥 업로드 순서에 따라 여러 주제의 포스트들이 뒤섞이게 됐다.

3. 비공개 포스트를 제외한 모든 글이 보인다. 계정을 하나 새로 파보기도 했지만 관리하기가 번거로웠다. 

### Obsidian으로

포스트를 편하게 관리할 수 있고, 간편하게 글을 쓰려면 뭐가 좋을까 고민하다가, **옵시디언**이 제일 먼저 떠올랐다.
당장 스벅 가서 클로드랑 얼른 구현했는데 만족스럽다. 자동으로 블로그에 반영이 되고, 폴더 기준으로 태그를 나눴다. 
코드블럭은 Tkdodo 블로그 스타일을 참고했다.
