마침 FE 지인이 NotionPresso라는 라이브러리를 만들고 있길래
'그럼 나는 Velog를 띄워야지' 생각해서 가져와서 스타일링하고 보여주었다.
그렇게 모든 구현을 마치고나서야 MDX에 대해 알게 되었다,, ㅠㅠ

### Velog 포스트를 띄워봤더니 생긴 문제들

1. 외부 게시물도 아니고, 내가 쓴 글 보여주려고 Tanstack Query를 쓰는 것부터 웃기다. 글을 빠른 속도로 보여주기 위해 클라이언트와 서버 로직을 분리하고, 무한 스크롤에서 포스트를 가져오려면  어쨌든 브라우저에서 외부 API 서버로 요청을 보내야하는데 CORS 문제를 겪게 된다. API Route를 거치게 되면 속도가 체감될 정도로 느려진다. rewrites로 프록시를 적용하는 등 온갖 생쑈를 해야 한다. 그래도 최적화 과정에서 배우는 건 많았다.

2. 포스트를 카테고리별로 분류하기 어렵다. 할수야 있지만 코드가 복잡해지고 번거롭다. 결국 그냥 업로드 순서에 따라 여러 주제의 포스트들이 뒤섞이게 된다.

3. 비공개 포스트를 제외한 모든 글들이 나온다. 계정을 하나 새로 파서 글을 써보기도 했지만, 관리 포인트가 늘어나서 지속하기 어렵다. 

### Obsidian으로 갈아타기

카테고리 분류, 메모장에 쓰듯 간편한 작성 등을 고려했을 때 옵시디언이 가장 먼저 떠올랐다.
어제 클로드와 함께 얼른 구현했는데 아주 만족스럽다. Obisidian에서 글을 쓰면 자동으로 블로그에 반영이 되고, 폴더 기준으로 태그가 나뉘도록 했다. 마크다운은 Tkdodo의 블로그를 참고해 스타일링 해줬다.

```typescript showLineNumbers {4-7}
import { RadioGroup, RadioGroupItem } from '@/components/radio'

  if (n <= 1) return n;
```

다른 분들의 개발 블로그를 둘러보면 posts 폴더에 들어있는 md 파일들을 볼 수 있는데, 이걸 IDE 상에서 쓰는 게 아니라, Obisidian같은 툴에서 프로젝트 폴더를 임포트해와서 파일을 생성해 쓰는 방식이었다.
